# 금융 상담 텍스트 위험도 3단계 라벨링 가이드

## Purpose

금융 고객상담 텍스트에서 개인정보 노출 가능성, 보안 민감도, 공격적 표현 가능성을 기준으로 위험도를 분류합니다. 본 가이드는 AI 안전성 모델 학습 데이터 구축 및 오탐 검수 업무를 가정한 기준표입니다.

## Label Definition

| Risk Level | Name | Definition | Example Type |
|---|---|---|---|
| Level 0 | 일반 상담 | 금융 상품, 이용 방법, 단순 조회 등 민감정보 노출 가능성이 낮은 상담 | 상품 안내, 이용 방법 문의, 일반 수수료 문의 |
| Level 1 | 주의 필요 | 계좌, 카드, 대출, 보험, 인증, 명의 등 금융 민감 맥락이 있으나 직접 식별정보가 확인되지 않는 상담 | 대출 연장 문의, 자동이체 확인, 보험 해지 요청 |
| Level 2 | 고위험 | 주민번호, 전화번호, 주소, 계좌번호, 카드번호, 인증번호 등 직접 식별정보 또는 강한 공격/협박/욕설 가능성이 있는 상담 | 인증번호 전달, 계좌번호 노출, 민원성 공격 표현 |

## Judgment Principles

1. 단어 하나만으로 판단하지 않고 상담 목적과 정보 노출 가능성을 함께 봅니다.
2. 직접 식별정보가 있는 경우 Level 2로 우선 분류합니다.
3. 직접 식별정보가 없어도 인증, 명의, 계좌, 대출, 보험금 청구처럼 금융 보안 사고와 연결될 수 있는 맥락은 Level 1 이상으로 검토합니다.
4. 상담 불만과 공격적 표현을 구분합니다. 단순 불만은 Level 1, 욕설·협박·차별·자해/타해 위협 가능성이 있으면 Level 2 후보로 봅니다.
5. 애매한 케이스는 별도 `review_required` 플래그를 두고 가이드라인 개선 후보로 기록합니다.

## Suggested Mapping From Existing Labels

AI Hub 라벨 필드를 직접 위험도 라벨로 간주하지 않고, 위험도 후보를 찾는 보조 신호로 사용합니다.

| Field | Use |
|---|---|
| `consulting_category` | 은행/보험/증권 도메인별 위험 패턴 비교 |
| `consulting_topic` | 대출, 자동이체, 거래내역, 보험금 청구 등 민감 주제 후보 탐지 |
| `task_category` | 핵심 금융 용어 포함 여부 확인 |
| `consulting_situation` | 일반문의/업무처리/민원응대별 검수 우선순위 설정 |
| `qa_topic` | 세부 상담 주제 기반 위험도 후보 분류 |
| `consulting_purpose` | 상담 의도 단위의 애매한 케이스 수집 |
| `core_financial_terms` | 개인정보/보안 관련 키워드 후보 탐지 |

## Review Queue Criteria

다음 조건에 해당하면 자동 분류 결과와 별개로 사람이 재검수합니다.

- `consulting_situation = '민원응대'`
- `core_financial_terms`에 인증, 계좌, 비밀번호, 주민, 카드, 명의 관련 표현이 포함됨
- `consulting_topic` 또는 `qa_topic`이 거래내역, 잔액조회, 자동이체, 대출, 보험금 청구와 관련됨
- 동일 source_id에서 여러 QA가 생성되어 문맥 분할 가능성이 높은 경우

## Portfolio Note

원문 상담 문장과 QA 본문은 외부에 공개하지 않습니다. 포트폴리오에는 라벨 기준표, 집계 통계, SQL 쿼리, 품질 점검 방식만 포함합니다.
